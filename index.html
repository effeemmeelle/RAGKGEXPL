Key Concepts

Embedding Models

Convert text → dense vectors (e.g. 768d). Similar meaning → similar direction in vector space. Popular: text-embedding-3BGEE5

Vector Databases

Store & index millions of vectors for fast approximate-nearest-neighbour (ANN) search. Popular: PineconeChromaWeaviateFAISS

Chunking Strategies

Fixed-size, sentence, paragraph, semantic, recursive. Chunk size vs overlap is a key trade-off. Too small = lost context. Too large = noise. See the Chunky dashboard ↗ for a live demo.

Compare & Combine

GraphRAG
💡 RAG and Knowledge Graphs are complementary. Combining them — sometimes called GraphRAG — gives you both the semantic flexibility of vector search and the precision of structured graph traversal.
Dimension 🟢 RAG 🟠 Knowledge Graph
Knowledge representationUnstructured text chunksTyped triples / property graph
Query mechanismApproximate vector similarityExact graph traversal (SPARQL/Cypher)
Multi-hop reasoning⚠ Weak — context window only✔ Strong — explicit path following
Handles fuzzy / semantic queries✔ Strong — embedding captures meaning✘ Weak — needs exact predicates
Build cost✔ Low — just embed text✘ High — NER + RE + curation
Explainability✘ Black-box similarity✔ Traceable provenance path
Handles new domains quickly✔ Just add docs✘ Must update ontology
Structured aggregation (counts, stats)✘ LLM must hallucinate✔ Native SPARQL aggregation
Best forOpen-domain Q&A, chat, summarizationEnterprise knowledge, compliance, recommendations
GraphRAG — Hybrid Architecture

How GraphRAG works

1. Vector search retrieves semantically similar chunks
2. KG traversal expands to related entities & facts
3. Both are merged into a rich context
4. LLM generates an answer grounded in both

When to choose what

Pure RAG: chatbots, docs Q&A, fast prototypes
Pure KG: knowledge management, drug discovery, compliance
GraphRAG: enterprise assistants, research tools, multi-hop over large corpora

Hybrid Query Flow
👤 User query: "Who influenced Eiffel's engineering?"
↓ embed query
🔢 Vector search → 3 text chunks about Eiffel
🕸️ KG traverse → Eiffel → influencedBy → Monier, Belanger
↓ merge context
🤖 LLM generates grounded, multi-hop answer
